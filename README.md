# Canvas Studio 🎨

**Canvas Studio for WormHole** — A modern web application to upload, manage, and search animated looping videos (Canvas) designed to integrate with the WormHole music application.

---

## 📋 Description

Canvas Studio provides a **Web API** and frontend dashboard that allows you to:

- 📤 **Upload animated videos** (MP4) associated with songs or entire albums
- 🔍 **Search for Canvases** by artist, album, and song
- 📋 **Manage** your collection (view catalog, copy direct URLs, and delete items)
- 🔌 **Integrate** with WormHole via a REST API
- 🎬 **Preview** canvases with smooth animations and playback inside the dashboard

It is the perfect companion to expand WormHole's Canvas library with custom or community-created video loops.

---

## 🚀 Key Features

### 1. Flexible Backend Architecture
- **Vercel Blob Storage Integration**: Built-in serverless backend (`/api/*`) that runs out of the box on Vercel.
- **Custom Worker Support**: Settings panel to connect the frontend to a custom Cloudflare Worker or external REST API base.

### 2. Modern Design System
- Sleek dark theme UI with fluid spacing.
- Responsive layout with notch compatibility (`viewport-fit=cover`) and floating action upload button (FAB) for mobile support.
- Content skeleton grids during loading states.

### 3. Full Album Mode
- Switch to "Full Album Mode" when uploading so a single canvas applies to all songs in the album.

### 4. Local Filtering & Instant Search
- Instant client-side search across the entire canvas catalog by artist, album, or song.

## 🛠 Tech Stack

- **Frontend**: HTML5, CSS3, JavaScript
- **Built-in Backend**: Vercel Serverless Functions (Node.js) + Vercel Blob Storage

## 🎯 Getting Started

### Using the Built-in Vercel Backend

Deploy the repository directly to Vercel. Out of the box, the frontend will automatically direct API requests to the `/api` folder.

#### Environment Variables for Vercel
- `BLOB_READ_WRITE_TOKEN` — Token for Vercel Blob Storage.
- `BLOB_STORE_ID` — Store ID (auto-generated by Vercel).

### Using a Custom Worker API

If you prefer to deploy your backend separately (e.g., on a Cloudflare Worker):
1. Click the **Server Connection** settings panel header.
2. Fill in the **Worker URL** and **Upload Token**.
3. Save connection settings.

## 📡 API Endpoints

The API base serves requests using the following endpoint query parameters (designed to match Cloudflare Worker style action mapping, rewritten automatically on Vercel):

### List Canvases
```
GET /api?action=list
```
Returns a list of all uploaded canvases.

### Search Canvas
```
GET /api/search?artist=X&album=Y&song=Z
```
Returns the best matching canvas for a song request.

### Upload Canvas
```
POST /api?action=upload
Header: Authorization: Bearer <token> (if custom worker base)
Body: multipart/form-data with artist, album, song (optional), and file.
```

### Delete Canvas
```
DELETE /api?action=delete
Header: Authorization: Bearer <token> (if custom worker base)
Body: { "id": "artist_album_song" }
```
